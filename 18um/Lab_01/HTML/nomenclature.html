How to Read DesignWare® Standard Cell Datasheets

Standard Cell Naming Convention

The DesignWare Standard Cells use a naming convention with the following fields:

(Function)(Variation)(Inversions)(OutputDrive)

Where:

Function is the abbreviated cell function

Variation is the number of inputs, functional configuration or clear/preset options

Inversions is an indicator of input or output pin inversions

OutputDrive is the output drive strength

Example:

For cell nand2a1:

nand is the function

2 is the number of inputs

a is the input inversion

1 is the drive strength

For cell fd2a1:
fd is the function

2 is the preset/clear option

a is the clock/data/output inversion

1 is the drive strength

Standard Cell Name Variation

The variation part of a Standard Cell name is different for different cell functions. For simple logical gates (e.g., AND, NAND, NOR, XOR, XNOR), the variation refers to the number of inputs. For complex gates (AND-OR, OR-AND), the variation refers to the logic configuration. For sequential cells, the variation refers to the existence or non-existence of preset and clear pins.

Output Drive

The output drive strength field indicates the relative size of the output transistors. The drive strength is an integer number. The strengths are approximately proportional. For example, a nand2a2 has about twice the drive strength of a nand2a1.

Setup and Hold Time

Setup and hold times are provided for sequential cells.
The given values are the worst-case values for any core cell input pin.
For example, the setup timing given for an "fdm1a1" cell is the worst setup time for D0, D1, and S.
